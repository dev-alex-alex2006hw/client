CLI Usage

API Modules

The planet-client package is organized as a collection of modules that roughly follow the structure of the HTTP API. The API provides access to individual scenes from a variety of providers and mosaics of those scenes.

planet-client/api/aois

Provides methods for listing and creating areas of interest (AOI), or stored geometries. AOIs allow you to query the scenes api without having to transmit what may be very large geometries with every request.

After creating an AOI, we'll respond with a json payload containing an id as well as the resulting geojson. The scenes api can then be queried using the aoi parameter and setting it to the id of your uploaded AOI.

var mod = require('planet-client/api/aois')

Functions

mod.get(id, options)

Obtains the metadata for a single uploaded geometry. The response includes the geometry inside the returned object's geojson field.

Arguments

id string

A mosaic identifier.
options Object

Options.
options.terminator function

A function that is called with a function that can be called back to terminate the request.

Returns

Promise.<Object>

A promise that resolves to aoi metadata or is rejected with any error. See the errors module for a list of the possible error types.

mod.list(id, options)

List all of your previously uploaded AOIs. Note that this does not include the geometry geojson for each AOI.

Arguments

id string

An AOI public id.
options Object

Options.
options.terminator function

A function that is called with a function that can be called back to terminate the request.

Returns

Promise.<Array.<Object>>

A promise that resolves to an array of AOI metadata or is rejected with any error. See the [errors module] (#module:planet-client/api/errors) for a list of the possible error types.

mod.create(name, file, options)

Creates a new AOI using uploaded json data.

Arguments

name String

The name of the AOI.
file Object

An object representing a file and it's contents.
file.name String

The name of the originally uploaded file.
file.contents String

The file's contents as a string.
options Object

Options.
options.terminator function

A function that is called with a function that can be called back to terminate the request.

Returns

Promise.<Array.<Object>>

A promise that resolves to an array of AOI metadata or is rejected with any error. See the [errors module] (#module:planet-client/api/errors) for a list of the possible error types.

planet-client/api/auth

Provides methods for authenticating with a Planet API account.

var mod = require('planet-client/api/auth')

Functions

mod.login(email, password)

Submit credentials for authentication. Upon successful authentication, a token representing the user will be stored for subsequent API requests.

Arguments

email string

The email associated with a Planet account.
password string

The password for a Planet account.

Returns

Promise

A promise that resolves on successful login and is rejected otherwise.

mod.logout()

Clear any stored credentials.

mod.setKey(key)

Set an API key to be used for subsequent requests. This is an alternative to submitting credentials with the login method. The stored key will be used for subsequent API requests.

Arguments

key string

An API key.

planet-client/api/errors

Includes specific Error types generated by API requests. When a request method returns a promise that is rejected, you can use instanceof to determine what type of error you have.

var mod = require('planet-client/api/errors')

Classes

mod.ResponseError

The base class for all response errors.

Properties

message string

A message providing details about the error.
response IncomingMessage

The server response.
body Object

Any parsed response body. For "expected" errors, this will be an object representing the JSON response body.
stack string

The stack trace for the error.

planet-client/api/page

Provides a utility for working with pages of search results.

var mod = require('planet-client/api/page')

Classes

mod.Page

A page of results. Pages do not need to be constructed directly, but are generated by methods that make API requests.

page.prev()

Get the previous page. If there is no previous page, prev will be null.

Returns

Promise.<Page>

The previous page.

page.next()

Get the next page. If there is no next page, next will be null.

Returns

Promise.<Page>

The next page.
